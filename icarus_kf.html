Project Icarus — State Estimation & Control

6-DoF quadrotor simulation with Kalman filter state estimation, complementary attitude filtering, and closed-loop PID trajectory tracking.

6-DoF Simulation Kalman Filter Complementary Filter PID Control Python · NumPy

Simulation

6-DoF quadrotor simulation — Kalman-filtered state estimates drive closed-loop PID control through a sequence of step waypoints.

System Overview

Project Icarus implements a full 6-DoF quadrotor simulation in Python, combining nonlinear flight dynamics with a sensor fusion stack and closed-loop control. The system models realistic GPS and IMU noise, fuses measurements with a Kalman filter and complementary filter, and tracks step waypoints using a PID controller operating on estimated states.

Key Results

  • Kalman filter converges rapidly from noisy GPS to within centimeters of true position.
  • Closed-loop controller tracks step waypoints in X, Y, and Z simultaneously.
  • Attitude remains stable through aggressive position transitions (<30° max tilt).
  • All four motor RPMs converge to steady-state hover after each maneuver.

Technical Architecture

Project Icarus implements a 6-state Kalman Filter to estimate translational position and velocity — [x, y, z, ẋ, ẏ, ż] — from noisy GPS and altimeter measurements. A complementary filter running on simulated IMU data provides high-frequency attitude estimates (roll, pitch, yaw), bypassing low-bandwidth sensor lag. These estimated states feed directly into a closed-loop PID controller for trajectory tracking.

The plots above demonstrate Kalman filter convergence from noisy measurements to stable state estimates in real time — the core requirement for any onboard GNC system where sensor fusion quality directly determines control performance.

Implementation Details

  • Language & Tools: Python, NumPy, Matplotlib, SciPy
  • Dynamics: Nonlinear 6-DoF quadrotor rigid body model
  • Estimator: 6-state linear Kalman filter (position + velocity)
  • Attitude: Complementary filter on simulated IMU data
  • Control: Cascade PID — outer position loop, inner attitude loop
  • Noise Model: Gaussian GPS noise, altimeter bias, IMU drift
