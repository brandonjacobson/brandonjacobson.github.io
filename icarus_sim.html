Project Icarus — Flight Simulations

Simulation work for Project Icarus, progressing from 1D PID altitude control to a full 6-DoF quadrotor simulation with Kalman filter state estimation and closed-loop trajectory tracking.

Python · NumPy · Matplotlib 6-DoF Dynamics Kalman Filter PID Control

6-DoF Simulation with Kalman Filter State Estimation

6-DoF quadrotor simulation — Kalman-filtered state estimates drive closed-loop PID control through a sequence of step waypoints.

The current simulation extends the 1D work to a full 6-DoF nonlinear quadrotor model. A 6-state Kalman filter fuses noisy GPS and altimeter measurements to estimate position and velocity, while a complementary filter handles attitude from simulated IMU data. These estimated states drive a cascade PID controller tracking step waypoints in X, Y, and Z simultaneously.

  • KF estimate converges to within centimeters of true position despite noisy GPS.
  • Closed-loop controller tracks step waypoints across all three axes.
  • Attitude remains stable through position transitions (<30° max tilt).
  • All four motor RPMs converge to steady-state hover after each waypoint.
Kalman Filter — True vs. Estimated vs. GPS

KF estimate (red) tracks true position (blue) closely despite scattered GPS readings (green ×). Altimeter data (orange) fuses with GPS for Z-axis estimation.

Full Telemetry — Position, Attitude & Motors

X/Y/Z position tracks step waypoints (dashed) with clean transient response. Euler angles settle to near-zero in hover. All four motor RPMs converge after each commanded waypoint.

A 6-state linear Kalman filter estimates [x, y, z, ẋ, ẏ, ż] from noisy GPS and altimeter measurements. A complementary filter running on simulated IMU data provides high-frequency attitude estimates (roll, pitch, yaw). These fused states feed a cascade PID controller — outer position loop commanding attitude targets, inner attitude loop commanding motor thrust — for closed-loop trajectory tracking in the nonlinear 6-DoF model.

Earlier Work — 1D Vertical Flight Simulation

The starting point for Icarus simulation work. A 1D model of vertical flight dynamics with PID altitude control, sensor noise modeling, and low-pass filtering — used to validate the control approach before moving to 6-DoF.

Key Highlights

  • Implemented vertical thrust simulation with autonomous altitude tracking.
  • Modeled noisy thrust and RPM measurements and evaluated controller robustness.
  • Applied low-pass filtering to stabilize control inputs under measurement noise.
  • Three-phase profile: ground → 10 m hover → soft touchdown.

Stack

  • Language & Tools: Python, NumPy, Matplotlib
  • Control: PID altitude control with noise-robust tuning
  • Modeling: Vertical thrust dynamics, noisy sensor measurements
  • Signal Processing: Low-pass filtering for smoother control inputs
